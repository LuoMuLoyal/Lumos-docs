- **API Contract Propagation**: The backend (`Lucent`) controllers and DTOs define the single source of truth. Changes trigger `pnpm export:openapi` to regenerate `openapi.json`, which the frontend (`Luminous`) consumes via a custom Dart script (`tool/regenerate_lucent_openapi.dart`) to update its generated API client.
- **Boundary Enforcement**: Strict separation between backend-owned data schedules (e.g., medicine reminders) and frontend-owned device-local execution (e.g., local notifications), documented in shared contract files like `reminder-contract.md`.
- **Independent Git Isolation**: Child modules (`Lucent`, `Luminous`) are separate git repositories managed from the workspace root using `git -C` commands, preventing cross-contamination of commit histories.
- **Shared Data Pipeline**: A dedicated ETL module (`DrugDataBase`) processes external pharmaceutical datasets (DrugBank, Chinese regulatory data) into a unified knowledge base consumed by the backend's medicine modules.
