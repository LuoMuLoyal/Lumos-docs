# 2026-06-22

## Tooling

- Migrated the frontend validation and git-hook toolchain from PowerShell to
  Dart: `tool/run_daily_checks.dart`, `tool/run_fullstack_checks.dart`,
  `tool/install_git_hooks.dart`, and `.githooks/pre-commit` / `pre-push` now
  call Dart directly instead of going through `.ps1` wrappers.
- Tightened the new Dart `pre-commit` hook to check formatting only for staged
  Dart files, so unrelated pre-existing format drift elsewhere in the repo does
  not block focused commits.
- Fixed the full-stack validation entry so `tool/run_fullstack_checks.dart`
  uses Lucent's current
  `pnpm test:runtime:start` / `pnpm test:runtime:stop` entrypoints instead of
  the removed PowerShell helper path.
- Fixed the Lucent Windows test-runtime helpers behind that wrapper so
  `pnpm test:runtime:start` now returns after background startup and
  `pnpm test:runtime:stop` kills the full runtime process tree instead of
  leaving orphaned `nest --watch` children behind.
- Pinned Android integration-test support dependencies in
  `android/build.gradle.kts` so Flutter's SDK-level dynamic `androidx.test`
  versions stop forcing flaky Maven metadata lookups during local full-stack
  runs.
- Updated frontend docs to match the current Lucent full-stack runtime startup
  command.
- Tightened the record full-stack lane so it waits for the create flow to pop
  back to the record page and then finds the created note by real timeline
  title content instead of assuming a fake fixed `recordId`.
- Added stable timeline row index keys so full-stack tests can open the newest
  record on a freshly prepared date without guessing Lucent-generated ids or
  depending on text rendering details.
- Fixed the sleep record edit flow for quick-choice / duration-only payloads:
  `RecordEditPage` now preserves an existing `durationMinutes` value when the
  record has no stored bedtime/wake-time range, so editing note/date/time no
  longer gets blocked by a false "invalid sleep duration" validation failure.
- Hardened the new quick-choice full-stack lane to read chip labels from the
  actual widget tree, tolerate duplicate text nodes inside a single timeline
  card, and keep asserting that timeline/detail/edit all show the saved
  occurredTime consistently.

## Medicine Safety Depth

- **Entity layer**: Added `SpecialPopulationConclusion` enum (5 levels:
  contraindicated / avoid / caution / consultClinician /
  insufficientInformation) and `specialPopulationConclusion` field to
  `MedicineRiskFinding`, enabling the UI to show a structured risk conclusion
  separate from raw evidence text.
- **Classifier**: Added conservative keyword-matching classifier in
  `MedicineRiskChecker` with 4-tier bilingual keyword tables
  (contraindicated > avoid > caution > consult), falling back to
  `insufficientInformation` when no keyword matches. Severity is now
  derived from the conclusion tier instead of being hardcoded per context
  (e.g. a pediatric text containing "禁用" now correctly maps to `high`
  rather than keeping the legacy `medium` default).
- **Two-layer UI display**: Updated `medicineRiskFindingTitle` to show the
  structured conclusion label (e.g. "禁用", "慎用") when available, and
  `medicineRiskFindingBody` to prepend the context label (e.g. "孕期 · 药品甲").
  Added 5 new l10n conclusion-label keys (zh/en).
- **Red-flag action layering**: Added per-rule action copy
  (`redFlagActionCopy`) with 3 explicit tiers: L1 "立即联系急救" for
  severeAllergy, L2 "停止自行用药/咨询医生" for pregnancyContraindication,
  L3 "高风险状态/建议线下确认" for informationGap. Updated
  `_RedFlagAlertRow` to render the action text with error icon between the
  description and resource button.
- **Red-flag honesty**: Removed `resourceId` from `informationGap` alerts
  (per plan: "不假装有线下资源闭环") — the action copy tells users to
  verify offline without linking to a fake action button.
- **Context leak fix**: Tightened special-population evidence reuse so the
  shared `pregnancyLactation` field no longer mirrors pregnancy-only wording
  into lactation findings or lactation-only wording into pregnancy findings.
  Explicitly combined wording still supports both contexts.
- **Red-flag compatibility**: Changed pregnancy/lactation contraindication
  escalation to prefer `specialPopulationConclusion ==
  contraindicated` while still falling back to legacy evidence keyword checks,
  so new structured findings and older fixtures both escalate correctly.
- **Boundary tests**: Added 30 new domain unit tests covering age thresholds
  (17/18/64/65/66/null), special-population missing-field boundaries
  (lactation, trying/postpartum, null states), allergy near-match false
  positives (aspirin≠acetaminophen, cephalosporin≠penicillin), unmapped
  drug matching, duplicate-ingredient disjoint guards, findings+coverage
  coexistence invariants, classifier tier mapping, and multi-keyword
  priority, plus 3 context-leak guards and 1 structured-red-flag compatibility
  case. Total medicine tests: 39 domain + 6 red-flag + 8 page = 53.
- **Rule matrix**: Filled the plan's rule-matrix draft into a 9-rule
  code-verified matrix with source line references, precise trigger
  conditions, and 9 documented critical boundary decisions for future
  review.
