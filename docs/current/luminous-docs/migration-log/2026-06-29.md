## OAuth Provider 扩展：Apple + QQ

### Lucent 后端

#### 新增 Provider

- **`apple-oauth.provider.ts`**（~185 行）：Apple Sign In 实现。使用 `jsonwebtoken` 校验 `identityToken`（JWKS→PEM→jwt.verify），一步完成签名/iss/aud/exp 验证。`fetchProfile(credential)` 接受 `{ identityToken, authorizationCode?, givenName?, familyName? }`。
- **`qq-oauth.provider.ts`**（~290 行）：QQ 互联 OAuth 2.0 三步流程。`buildAuthorizeUrl` 生成授权 URL，`fetchProfile()` 串行调用 token → openid → userinfo API。处理 QQ 特有的 query-string / JSONP 响应格式。
- **`wechat-base-oauth.provider.ts`**（~100 行）：从两个微信 Provider 抽取共用基类 `WechatBaseOAuthProvider`：`fetchWechat<T>()`、`isWechatError()`、`toJsonValue()`、WeChat API 接口定义和常量。

#### 重构

- **`IOAuthProvider` → `OAuthProvider`**：去掉 `I` 前缀，与项目命名风格统一。`oauth.types.ts` 中 union type 同步改为 `OAuthProviderName` 以消除命名冲突。
- **`wechat-web-oauth.provider.ts`** / **`wechat-mobile-oauth.provider.ts`**：`implements OAuthProvider, OnModuleInit` + `extends WechatBaseOAuthProvider`，各减少 ~70 行重复代码。
- **`fetchProfile(credential: Record<string, unknown>)`**：所有 Provider 统一签名，`credential.code` / `credential['identityToken']` 等由各实现自行提取。

#### Bug 修复 — OAuth State 冲突

- `auth-oauth-state.service.ts`：`OAuthStateEntry.provider` 从硬编码 `typeof OAUTH_PROVIDER_WECHAT_WEB` 改为 `OAuthProviderName`。

- `normalizeCallbackUri` 新增 `providerCallbackPath()` 按 provider 校验回调路径（`/login/oauth/wechat` vs `/login/oauth/qq`）。

#### 配置与合约

- **`env-keys.enum.ts`**：新增 `APPLE_APP_ID`、`QQ_APP_ID`、`QQ_APP_SECRET`、`QQ_REDIRECT_URI`。
- **`environment.validation.ts`**：`EnvironmentVariables` 接口新增对应字段。
