## Record/Report UI/UX responsive pass

Removed the redundant "Today overview" block from the Record mobile dashboard and made the Report page finding cards plus several grid/hero dimensions responsive.

### What changed

- Removed `RecordMobileOverview` from the Record mobile dashboard (`record_dashboard_view.dart`).
- Deleted `record_mobile_overview.dart` and removed the overview-only helpers from `record_dashboard_tokens.dart`.
- Removed now-unused ARB keys `recordTodayOverviewTitle`, `recordTodayOverviewEvents`, `recordTodayOverviewEventCount` from `app_en.arb`/`app_zh.arb` and regenerated l10n files.
- Updated `test/record/record_page_test.dart` to stop expecting the removed `record-today-overview` key and the "饮水进度" summary text.
- Added `lib/core/design/app_responsive_sizing.dart` with helpers for card width, sidebar width, grid column count, and width/height scaling with min/max clamps.
- Widened the Report findings card: replaced fixed `width: 238` with `AppResponsiveSizing.cardWidth(context)` (screen-width fraction clamped to 260–320).
- Made Record desktop sidebars responsive: `300`/`330` fixed widths now use `AppResponsiveSizing.sidebarWidth(context)`.
- Removed the fixed `height: 92` on Record quick-entry rows so they size to content.
- Made Report grid `mainAxisExtent` values responsive by breakpoint:
  - metrics cards
  - export cards
  - pattern cards
- Scaled Report score hero: score font and the hero icon circle now scale with available width.
- Scaled Report trend chart placeholder height and right padding with screen dimensions.

### Why

The Record "Today overview" duplicated information already visible in the timeline and quick-action counts, adding vertical bulk without clear utility. The Report finding card felt cramped on modern phone widths. Several grids and hero elements used fixed dimensions that broke layout or looked mismatched on larger phones, tablets, and desktop windows.

### Verification

- `flutter analyze` passes with no issues.
- `flutter test` passes (record and report suites validated, full suite run).

## Report period selector

Replaced the two placeholder `ReportPeriodPill` toasts on the Report page with a real period selector backed by Lucent's report dashboard API.

### What changed

- **Lucent backend**
  - Extended `ReportDashboardQueryDto` and `GenerateReportSummaryDto` with a new `custom` range plus optional `startDate`/`endDate` query parameters.
  - Updated `ReportsContextService` to resolve custom date ranges and validate start/end date ordering and future-date constraints.
  - Updated `ReportsAiSummaryService` to pass custom range dates through to the dashboard context builder.
  - Added/updated controller and context service specs for the new ranges.
  - Exported an updated `docs/openapi.json`.

- **Luminous generated client**
  - Regenerated `packages/lucent_openapi` so `ReportsApi.reportsControllerGetDashboardV1` accepts `range`, `startDate`, and `endDate`.

- **Luminous frontend**
  - Added `ReportDashboardRange` enum (`last7Days`, `last30Days`, `custom`) and added `range`, `startDate`, `endDate` to the `ReportDashboard` entity.
  - Added `ReportDashboardQuery` value object to carry a range plus optional custom `startDate`/`endDate`.
  - Converted `reportDashboardProvider` to a `FutureProvider.family` keyed by `ReportDashboardQuery` and added `reportDashboardSelectedQueryProvider`.
  - Wired `ReportRemoteDataSource`, `LucentReportRepository`, and `MockReportRepository` to accept and pass the full query (presets skip dates; `custom` sends ISO date strings).
  - Replaced the hard-coded date label in `ReportTopBar` with the real `dashboard.startDate`/`endDate` formatted range.
  - Changed `ReportPeriodPill` to display the current range and open a bottom-sheet picker; both the top-bar pill and the trend-section pill now stay in sync.
  - Added a shared `ReportRangePickerSheet` for selecting between `last_7_days`, `last_30_days`, and `custom`. Selecting `custom` opens the platform date-range picker and fetches the dashboard for the chosen dates.
  - Added `reportRangePickerTitle` and `reportRangeCustom` ARB keys; removed now-unused `reportWeekDateRange` and `reportPeriodThisWeek`.
  - Added `report_ui_formatters.dart` for locale-aware date-range formatting.
  - Updated report data-source, repository, and page tests for the new query parameter and entity fields.

- `cd Luminous && flutter pub get && flutter gen-l10n && dart run build_runner build && flutter analyze && flutter test`
