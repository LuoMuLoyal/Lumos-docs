## Sleep Contract + Persistence MVP

### Lucent Backend

- Added `sleep` to `DailyRecordKind` Prisma enum with migration `20260613140000_add_sleep_to_daily_record_kind`.
- Added optional `payload` field to `CreateDailyRecordDto`, `UpdateDailyRecordDto`, and `DailyRecordItemDto`.
- `DailyRecordsMapperService` passes through `payload` in `toItem` and `toRecordUpdateData`.
- `DailyRecordsService.create` writes `payload` to database.
- `TodayAnalysisContextService` reads real sleep data from payload (durationMinutes, quality, startAt, endAt, deepMinutes, lightMinutes, remMinutes).
- `ReportsContextService.buildSleepSeries` computes real sleep hours from persisted `durationMinutes`.
- `ReportsComputationService.buildSleepMetric` computes real sleep status (>=7h good, >=5h stable, <5h needs_attention).

- `record_type_mapping.dart`: `RecordEntryType.sleep` ↔ `DailyRecordKind.sleep` bidirectional mapping.
- Unlocked sleep quick action and filter in `LucentRecordRepository` and `MockRecordRepository` (removed `locked: true`).
- Added sleep to timeline entry switch (icon, accent, titleKey).
- `daily_record_form_fields.dart`: sleep form rules (showValue + showUnit), kind label, value label, added to `activeDailyRecordKinds`.
- `record_detail.dart`: sleep kind icon and label.
- `LucentTodayRepository`: reads sleep payload from daily record summary, formats duration as `Xh` label.
- Added `recordCreateValueSleep` l10n key (EN: "Duration (e.g. 8h)", ZH: "时长（如 8h）").
- Regenerated `packages/lucent_openapi` client.

## Sleep Follow-up Hardening

### Lucent Backend

- `DailyRecordsGuardService.ensureOwnedByUser` now returns the existing record kind and payload snapshot for update-time validation.
- `DailyRecordsService` now rejects invalid sleep records both on create and on update, based on the final post-patch state instead of only the patch fragment.
- `ReportsContextService` now orders same-day sleep records by `occurredAt` and `createdAt` so the most recently created same-day sleep entry wins deterministically.

### Luminous Frontend

- Record create/edit now parse sleep durations from plain numeric input or `xh` input such as `7.5` and `8h`.
- Record create/edit now block invalid sleep durations before save and show a localized validation toast.
- Record create/edit/detail delete flows now invalidate `reportDashboardProvider` so Report refreshes after sleep changes.
- Sleep values are normalized before save so timeline/detail rows do not render duplicated unit text like `8h h`.

## Full-Stack Lane Hardening

- Fixed the failing full-stack sleep lane by regenerating `packages/lucent_openapi` after confirming the generated `DailyRecordKind` enum had gone stale and was missing `sleep`.
