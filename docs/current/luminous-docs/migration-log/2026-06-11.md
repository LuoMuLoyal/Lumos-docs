### Full-Stack Mobile E2E Lane - Phase 5 Execution Decision

- Decision: keep the first real lane local/manual for now. Do not add it to GitHub Actions yet.
- Reason:
  - `Luminous` CI is still a simple Ubuntu validation workflow (`flutter analyze` + `flutter test`) and has no emulator runtime today.
  - `Lucent` and `Luminous` are separate repositories, so this lane would require cross-repo orchestration instead of a straightforward same-repo CI step.
  - The lane currently depends on a Lucent test runtime plus Docker-backed test Postgres. Forcing Android emulator + Docker + cross-repo boot into CI now would add maintenance and flake risk before we even have a second real lane.
- Owner command is now documented in `docs/Next_Plan.md`.
- Expected usage:
  - rerun locally before merging auth/session changes
  - rerun locally before merging `/api/v1/user/daily-records*` contract or UI changes
  - rerun locally before claiming Record CRUD stability in a mobile test build
- With the decision recorded, the full-stack E2E lane plan is complete and the next implementation target can move away from E2E infrastructure work.
