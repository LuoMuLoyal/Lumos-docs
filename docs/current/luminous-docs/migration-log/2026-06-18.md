## AI Chat 会话持久化首轮打通

- `/assistant` 现在会在已登录状态下调用 Lucent `GET /api/v1/user/assistant/latest`，恢复当前用户最新一条持久化对话，而不是每次重新打开都回到空白临时会话。
- AI chat controller 新增会话恢复状态，与 capabilities 加载拆开处理，避免“能力先返回、历史后覆盖”的竞态；会话恢复失败时页面会显示明确错误和重试入口。
- 清空动作不再只是本地 wipe，而是调用 Lucent `POST /api/v1/user/assistant/latest/clear` 归档最新 active conversation；清空失败时保留当前消息，不制造前后端状态分叉。
- SSE 最终结果现在会读取 `conversationId`，为后续多会话扩展保留真实会话标识。
- 修复失败重试重复追加用户消息的问题：如果失败前的最后一条就是待重试用户消息，retry 会复用该消息而不是再插入一遍。
- 调整禁用态行为：如果用户已关闭 AI chat 但存在已恢复历史，对话内容仍然可见，只禁用继续发送；只有“无历史且不可发消息”时才显示空禁用占位。
- AI chat widget tests 新增持久化恢复、后端清空、retry 不重复、禁用态保留历史等覆盖。
- `/assistant` 右上角现在固定显示加号入口；点击后会开始一条新对话，当前实现会先归档当前 active conversation，再把页面重置到新的空白会话状态。

## AI Chat 历史总结与持久化记忆纠偏

- assistant 的“历史 AI”定义已经纠偏：不再把聊天历史或旧助手消息当作历史 AI，总线改成读取真正持久化的 `Today/Report` AI 总结历史。
- Lucent 新增并已最终收口为 `assistant_summary_histories` 的历史总结持久化模型；Today AI 总结和 Report AI 总结在每次生成后都会写入这张表，后续 assistant 读取的是同一份服务端真相，而不是客户端缓存。
- Lucent AI chat 读工具清单已把 `get_recent_ai_chat_messages / get_recent_ai_advice_from_chat` 替换成 `get_recent_today_summaries / get_recent_report_summaries`。
- assistant 会话持久化与“跨会话记忆”现在明确分开：会话持久化继续用于恢复、最近列表和切换；跨会话记忆改成 `assistantMemoryEnabled` 独立设置，默认关闭。
- 当 `assistantMemoryEnabled` 打开时，Lucent 只会在“新对话起点”注入一段受限的持久化记忆块，而不会在每次继续追问时重复塞回旧上下文，避免“新对话”名义上清空、实际上没清空。
- `/assistant` 控制面板现在新增“启用持久化记忆”开关，和“启用 AI 对话”分开显示，避免用户把聊天可用性和跨会话记忆混成一个概念。
- AI chat 工具标签已同步更新为“今日历史总结 / 报告历史总结”，不再显示旧的最近聊天/历史建议标签。
